Section 01

The Perceptron

The atom of all neural networks. A perceptron takes weighted inputs, sums them, adds a bias, and passes through an activation. Everything else is just scaling this up.

Single Neuron

A single neuron: weighted sum → add bias → apply activation → output
Forward Pass — Single Neuron
z = wᵀx + b  =  Σᵢ wᵢxᵢ + b
ŷ = σ(z)  where  σ  is an activation function
Python — Neuron from scratch
import numpy as np

class Neuron:
    def __init__(self, n_inputs):
        # He initialisation: good for ReLU networks
        self.w = np.random.randn(n_inputs) * np.sqrt(2 / n_inputs)
        self.b = 0.0

    def forward(self, x):
        self.x = x                        # cache for backprop
        self.z = np.dot(self.w, x) + self.b  # linear combination
        self.a = np.maximum(0, self.z)   # ReLU activation
        return self.a

    def backward(self, da):
        # Gradient of ReLU: 1 if z>0, else 0
        dz = da * (self.z > 0).astype(float)
        self.dw = dz * self.x
        self.db = dz
        return dz * self.w  # grad w.r.t. input

neuron = Neuron(n_inputs=3)
x = np.array([0.5, -0.3, 1.2])
output = neuron.forward(x)   # ReLU(wᵀx + b)
Section 02

Multi-Layer Perceptron (MLP)

Stack layers of neurons. The hidden layers learn increasingly abstract representations. This is what "deep" in deep learning means.

Network Architecture

MLP: 4 → 5 → 4 → 3. Every layer fully connected to the next. Each hidden neuron learns a different feature combination.
Layer-wise Forward Pass
Z⁽ˡ⁾ = W⁽ˡ⁾ A⁽ˡ⁻¹⁾ + b⁽ˡ⁾    (linear)
A⁽ˡ⁾ = σ(Z⁽ˡ⁾)    (activation)
W⁽ˡ⁾ ∈ ℝ^{hₗ × hₗ₋₁}   b⁽ˡ⁾ ∈ ℝ^{hₗ}
★ Universal Approximation Theorem

An MLP with a single hidden layer and a non-linear activation can approximate any continuous function to arbitrary precision — given enough neurons. This is why neural networks are so powerful. In practice, depth is more efficient than width for complex functions.

Without non-linear activations, any deep network collapses to a single linear transform. Activations are what give networks their expressive power.

Section 04

Backpropagation

Backprop is automatic differentiation applied to neural networks — it efficiently computes the gradient of the loss with respect to every weight using the chain rule.

The Chain Rule

Chain Rule — Foundation of Backprop
∂L/∂w = ∂L/∂a · ∂a/∂z · ∂z/∂w
 = δ · σ'(z) · x

Full Backprop Pass

Backprop Equations (layer l)
δ⁽ˡ⁾ = (W⁽ˡ⁺¹⁾)ᵀ δ⁽ˡ⁺¹⁾ ⊙ σ'(Z⁽ˡ⁾)    (error propagation)
∂L/∂W⁽ˡ⁾ = δ⁽ˡ⁾ (A⁽ˡ⁻¹⁾)ᵀ / n    (weight gradient)
∂L/∂b⁽ˡ⁾ = mean(δ⁽ˡ⁾, axis=batch)    (bias gradient)
Forward pass computes activations (blue →). Backward pass propagates gradients (amber ←). Each node needs to store its forward values for gradient computation.
Python — Backprop from scratch (2-layer net)
import numpy as np

def relu(z):        return np.maximum(0, z)
def relu_grad(z):   return (z > 0).astype(float)
def softmax(z):
    e = np.exp(z - z.max(axis=1, keepdims=True))
    return e / e.sum(axis=1, keepdims=True)

# ── FORWARD PASS ──
def forward(X, W1, b1, W2, b2):
    Z1 = X @ W1.T + b1    # (n, h1)
    A1 = relu(Z1)          # (n, h1)
    Z2 = A1 @ W2.T + b2   # (n, k)
    A2 = softmax(Z2)       # (n, k)
    cache = (X, Z1, A1, Z2, A2)
    loss  = -np.log(A2[np.arange(len(X)), y] + 1e-9).mean()
    return loss, cache

# ── BACKWARD PASS ──
def backward(cache, W2, y):
    X, Z1, A1, Z2, A2 = cache
    n = X.shape[0]

    # Output layer gradient (softmax + cross-entropy gradient is clean)
    dZ2 = A2.copy()
    dZ2[np.arange(n), y] -= 1
    dZ2 /= n              # (n, k)

    dW2 = dZ2.T @ A1     # (k, h1)
    db2 = dZ2.sum(axis=0)# (k,)

    # Hidden layer gradient
    dA1 = dZ2 @ W2        # (n, h1)
    dZ1 = dA1 * relu_grad(Z1)  # element-wise gate

    dW1 = dZ1.T @ X      # (h1, d)
    db1 = dZ1.sum(axis=0)# (h1,)
    return dW1, db1, dW2, db2
// PyTorch autograd — no manual backprop needed

PyTorch builds a dynamic computation graph during the forward pass. Call loss.backward() and all gradients are automatically computed. optimizer.step() then updates weights. You never write backprop by hand in production.

Section 05

Loss Functions

The loss function defines what the network is optimising. Choosing the wrong loss is choosing the wrong problem to solve.

TaskLossFormulaPyTorch
Binary classificationBinary Cross-Entropy−(y log ŷ + (1−y)log(1−ŷ))nn.BCEWithLogitsLoss()
Multi-classCross-Entropy−Σ yₖ log ŷₖnn.CrossEntropyLoss()
RegressionMSEmean((y−ŷ)²)nn.MSELoss()
Regression (robust)Huber / MAEsmooth L1nn.HuberLoss()
Ranking / embeddingsContrastivemax(0, m−‖pos‖+‖neg‖)nn.TripletMarginLoss()
⚠ Never use nn.Sigmoid() + nn.BCELoss()

Always use nn.BCEWithLogitsLoss() — it combines sigmoid and BCE in one numerically stable operation. Similarly use nn.CrossEntropyLoss() (includes softmax) not nn.NLLLoss(nn.LogSoftmax(x)).

Section 06

Optimisers

The optimiser decides how to update weights given gradients. Choosing the right one — and tuning its learning rate — is one of the highest-leverage decisions in DL.

Interactive: Optimiser Convergence Race

Simple but noisy. Needs careful lr tuning. With momentum it's competitive.
SGD + Momentum
v ← βv − α∇L; w ← w+v
Builds velocity in consistent directions. Damps oscillations. β=0.9 typical.
RMSProp
s ← βs+(1−β)g²; w ← w−α·g/√s
Adaptive lr per parameter. Good for RNNs. Divides by gradient magnitude.
Adam
m=β₁m+(1−β₁)g; v=β₂v+(1−β₂)g²
Combines momentum + RMSProp. Default choice. β₁=0.9, β₂=0.999.
AdamW
Adam + weight decay decoupled
Adam but with proper L2 regularisation. Use for transformers. Default in HuggingFace.
LR Schedulers
α(t) = f(epoch)
Cosine annealing, warmup + decay, ReduceLROnPlateau. Always use one.
Python — Optimisers in PyTorch
import torch
import torch.optim as optim

# AdamW — best default for most tasks
optimizer = optim.AdamW(model.parameters(),
                        lr=3e-4,
                        weight_decay=1e-2,
                        betas=(0.9, 0.999))

# SGD + Nesterov momentum — often beats Adam for CNNs
optimizer = optim.SGD(model.parameters(), lr=0.1,
                      momentum=0.9, nesterov=True,
                      weight_decay=5e-4)

# ── Learning rate schedulers ──
# Cosine annealing (best for most tasks)
scheduler = optim.lr_scheduler.CosineAnnealingLR(
    optimizer, T_max=num_epochs, eta_min=1e-6)

# Linear warmup + cosine decay (best for transformers)
from transformers import get_cosine_schedule_with_warmup
scheduler = get_cosine_schedule_with_warmup(
    optimizer, num_warmup_steps=100,
    num_training_steps=num_epochs * len(train_loader))

# ReduceLROnPlateau — reduce when val loss stagnates
scheduler = optim.lr_scheduler.ReduceLROnPlateau(
    optimizer, mode='min', patience=5, factor=0.5)

# Step in train loop:
scheduler.step()               # after each epoch
scheduler.step(val_loss)       # for ReduceLROnPlateau

Regularisation in DL

Deep networks have millions of parameters and overfit easily. These techniques are essential to generalise beyond the training set.

TechniqueHow it worksWhen to use
DropoutRandomly zero p% of neurons during training → ensemble of sub-networksFC layers; p=0.1–0.5
Batch NormNormalise activations per mini-batch → stable training, allows higher LRAfter linear/conv, before activation
Layer NormNormalise across features per sample → batch-size independentTransformers, RNNs
Weight Decay (L2)Add λΣwᵢ² to loss → shrinks weights toward zeroAlways; use AdamW not Adam
Data AugmentationCreate new training examples via transforms → increases effective dataComputer vision always
Early StoppingStop when val loss stops improving → prevents overfittingAlways monitor val loss
Label SmoothingSoften hard 0/1 targets to 0.9/0.05 → calibrated probabilitiesClassification; ε=0.1
Python — Regularisation in PyTorch
import torch.nn as nn

class RegularisedMLP(nn.Module):
    def __init__(self, in_dim, hidden, out_dim, dropout=0.3):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(in_dim, hidden),
            nn.BatchNorm1d(hidden),    # normalise activations
            nn.ReLU(),
            nn.Dropout(p=dropout),    # randomly zero neurons

            nn.Linear(hidden, hidden),
            nn.BatchNorm1d(hidden),
            nn.ReLU(),
            nn.Dropout(p=dropout),

            nn.Linear(hidden, out_dim)  # no dropout on output
        )
        self._init_weights()

    def _init_weights(self):
        for m in self.modules():
            if isinstance(m, nn.Linear):
                nn.init.kaiming_normal_(m.weight, nonlinearity='relu')
                nn.init.zeros_(m.bias)

    def forward(self, x):
        return self.net(x)

# Label smoothing loss
criterion = nn.CrossEntropyLoss(label_smoothing=0.1)

# Gradient clipping — prevents explosions
nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)

Building Models with nn.Module

Python — nn.Module
import torch.nn as nn
import torch.nn.functional as F

class MLP(nn.Module):
    def __init__(self, d_in, hidden, d_out, dropout=0.2):
        super().__init__()
        self.fc1 = nn.Linear(d_in, hidden)
        self.fc2 = nn.Linear(hidden, hidden)
        self.fc3 = nn.Linear(hidden, d_out)
        self.bn1 = nn.BatchNorm1d(hidden)
        self.bn2 = nn.BatchNorm1d(hidden)
        self.drop = nn.Dropout(dropout)

    def forward(self, x):
        x = self.drop(F.relu(self.bn1(self.fc1(x))))
        x = self.drop(F.relu(self.bn2(self.fc2(x))))
        return self.fc3(x)   # raw logits — let loss handle softmax

model = MLP(128, 256, 10).to(device)
print(model)
print(f"Parameters: {sum(p.numel() for p in model.parameters()):,}")
Section 09

Training Pipeline

A complete, production-quality training loop with dataset handling, validation, checkpointing, and monitoring.

Python — Complete Training Pipeline
import torch, torch.nn as nn
from torch.utils.data import DataLoader, TensorDataset
import numpy as np

# ─── 1. Dataset & DataLoader ───
X_t = torch.tensor(X_train, dtype=torch.float32)
y_t = torch.tensor(y_train, dtype=torch.long)
train_ds = TensorDataset(X_t, y_t)
train_dl = DataLoader(train_ds, batch_size=64, shuffle=True,
                       num_workers=2, pin_memory=True)

# ─── 2. Model, loss, optimiser ───
model     = MLP(X_train.shape[1], 256, num_classes).to(device)
criterion = nn.CrossEntropyLoss(label_smoothing=0.1)
optimizer = torch.optim.AdamW(model.parameters(), lr=3e-4, weight_decay=1e-2)
scheduler = torch.optim.lr_scheduler.CosineAnnealingLR(optimizer, T_max=50)

# ─── 3. Training loop ───
best_val, patience_cnt = np.inf, 0
history = {'train_loss':[], 'val_loss':[], 'val_acc':[]}

for epoch in range(1, 51):
    # ── Train ──
    model.train()    # enables dropout, batch norm in train mode
    train_loss = 0
    for xb, yb in train_dl:
        xb, yb = xb.to(device), yb.to(device)
        optimizer.zero_grad()           # clear old gradients
        logits = model(xb)
        loss   = criterion(logits, yb)
        loss.backward()               # compute gradients
        nn.utils.clip_grad_norm_(model.parameters(), 1.0)
        optimizer.step()             # update weights
        train_loss += loss.item()

    # ── Validate ──
    model.eval()     # disables dropout, uses batch norm running stats
    val_loss, correct, total = 0, 0, 0
    with torch.no_grad():
        for xb, yb in val_dl:
            xb, yb = xb.to(device), yb.to(device)
            logits = model(xb)
            val_loss += criterion(logits, yb).item()
            correct  += (logits.argmax(1) == yb).sum().item()
            total    += len(yb)

    val_acc = correct / total
    scheduler.step()

    # ── Early stopping + checkpointing ──
    if val_loss < best_val:
        best_val = val_loss; patience_cnt = 0
        torch.save(model.state_dict(), 'best_model.pt')
    else:
        patience_cnt += 1
        if patience_cnt >= 10: break   # early stop

    print(f"E{epoch:02d} | train={train_loss/len(train_dl):.4f} "
          f"| val={val_loss/len(val_dl):.4f} | acc={val_acc:.4f}")

# ─── 4. Load best & evaluate ───
model.load_state_dict(torch.load('best_model.pt'))
model.eval()
✓ Training checklist

✅ Always call model.train() before training loop and model.eval() before validation.
✅ Always call optimizer.zero_grad() before loss.backward().
✅ Use torch.no_grad() during evaluation — 40% speed boost, no memory waste.
✅ Use pin_memory=True in DataLoader if using GPU — faster CPU→GPU transfer.
✅ Save model.state_dict(), not the full model — more portable.

Architecture Zoo

Key architectural innovations that shaped modern deep learning — from the original MLP to transformers. Each solved a specific failure mode of its predecessors.

1958 — 1986
MLP + Backpropagation
Multi-layer perceptrons trained via gradient descent. Solved XOR. Foundation of all DL.
1998
LeNet — CNNs for Vision
Convolutional layers for spatial feature extraction. Trained on MNIST with shared weights.
2012
AlexNet — Deep CNN on GPU
Won ImageNet by 10 points. Introduced ReLU, dropout, GPU training. Started the DL revolution.
2014
GAN — Generative Adversarial Networks
Generator + Discriminator adversarial game. Learns to generate realistic images from noise.
2015
ResNet — Skip Connections
Residual connections F(x)+x. Solved vanishing gradients for 100+ layer networks. Still widely used.
2017
Transformer — Attention is All You Need
Self-attention mechanism replaces RNNs. Parallelisable. Foundation of BERT, GPT, Vision Transformers.
2018–now
Foundation Models — BERT, GPT, CLIP, DALL·E
Pre-train on massive data, fine-tune on task. Transfer learning at scale. LLMs are the dominant paradigm.

Residual Block — The ResNet Innovation

Residual block: output = F(x) + x. The skip connection ensures gradients can flow directly to early layers, enabling 100+ layer networks.
Python — ResNet Block in PyTorch
class ResidualBlock(nn.Module):
    def __init__(self, dim):
        super().__init__()
        self.block = nn.Sequential(
            nn.Linear(dim, dim),
            nn.BatchNorm1d(dim),
            nn.ReLU(),
            nn.Linear(dim, dim),
            nn.BatchNorm1d(dim),
        )
        self.relu = nn.ReLU()

    def forward(self, x):
        return self.relu(self.block(x) + x)  # F(x) + x

class DeepResNet(nn.Module):
    def __init__(self, d_in, hidden, n_blocks, d_out):
        super().__init__()
        self.embed  = nn.Linear(d_in, hidden)
        self.blocks = nn.Sequential(
            *[ResidualBlock(hidden) for _ in range(n_blocks)]
        )
        self.head = nn.Linear(hidden, d_out)

    def forward(self, x):
        x = F.relu(self.embed(x))
        x = self.blocks(x)
        return self.head(x)

model = DeepResNet(d_in=128, hidden=256, n_blocks=8, d_out=10)
