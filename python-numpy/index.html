Section 01

Python Basics

Everything that can be represented in a computer has a type. Python's type system is dynamic but strongly typed — implicit coercion is minimal.

Variables & Built-in Types

Python variables are names bound to objects. The name does not have a type; the object it points to does. Every assignment creates (or rebinds) a reference.

Python
# --- Core types ---
x   = 42            # int   — arbitrary precision integer
pi  = 3.14159       # float — 64-bit IEEE 754 double
c   = 2+3j          # complex
s   = "hello"       # str   — immutable Unicode sequence
b   = True          # bool  — subclass of int (True==1, False==0)
n   = None          # NoneType — the null sentinel

# --- Type checks ---
type(x)             # <class 'int'>
isinstance(x, int)  # True

# --- Dynamic rebinding ---
x = "now a string"  # perfectly legal — name rebinds to new object

# --- Multiple assignment ---
a, b, c = 1, 2, 3
a, *rest = [10, 20, 30, 40]   # rest = [20, 30, 40]
𝓔 Integer arithmetic note

Python integers have arbitrary precision — 2 ** 1000 works without overflow. Floats use IEEE 754 double precision: 64 bits (1 sign, 11 exponent, 52 mantissa), giving ~15–17 significant decimal digits.

Section 02

Data Structures

Python's built-in collections are implemented in C and highly optimised. Choosing the right one is essential for ML code performance.

📋
list

Ordered, mutable, heterogeneous. O(1) append, O(n) insert.

📚
dict

Hash map. O(1) average lookup. Insertion-ordered (3.7+).

🔗
set

Hash set. O(1) membership test. No duplicates, unordered.

🔒
tuple

Immutable sequence. Faster iteration, hashable (if elements are).

Lists — the workhorse

Python
nums = [3, 1, 4, 1, 5, 9, 2]

# Slicing  [start : stop : step]
nums[1:4]      # [1, 4, 1]
nums[::2]      # [3, 4, 5, 2]  — every other
nums[::-1]     # [2, 9, 5, 1, 4, 1, 3] — reversed

# Mutation
nums.append(6)        # add to end
nums.insert(0, 0)     # insert at index 0
nums.pop()            # remove & return last element
nums.sort(reverse=True)  # in-place descending sort

# List comprehension — preferred over map/filter
squares  = [x**2 for x in range(10)]
evens    = [x    for x in range(20) if x % 2 == 0]
matrix   = [[0]*3 for _ in range(3)]  # 3×3 zero matrix

Dictionaries & Sets

Python
# --- dict ---
model_info = {"name": "ResNet", "layers": 50, "acc": 0.94}
model_info["params"] = "25M"        # add key
model_info.get("lr", 0.001)        # safe lookup with default
model_info.items()                  # dict_items view for iteration

# dict comprehension
word_len = {w: len(w) for w in ["cat", "elephant", "dog"]}

# --- set ---
a = {1, 2, 3, 4}
b = {3, 4, 5, 6}
a & b   # {3, 4}   — intersection
a | b   # {1,2,3,4,5,6} — union
a - b   # {1, 2}  — difference
a ^ b   # {1,2,5,6} — symmetric difference

# Remove duplicates from list (order lost)
unique = list(set([1,2,2,3,3,4]))
✓ ML Tip

Use defaultdict(list) from collections to group training samples by class label without KeyError. Use Counter to quickly count class frequencies and check for imbalance.

Functions & Functional Tools

Functions are first-class objects in Python — they can be passed as arguments, returned from other functions, and stored in data structures.

Defining Functions

Python
def sigmoid(z: float) -> float:
    """Squash any real number into (0, 1).  σ(z) = 1 / (1 + e^{-z})"""
    import math
    return 1 / (1 + math.exp(-z))

sigmoid(0)    # 0.5
sigmoid(2)    # 0.8808

# --- *args and **kwargs ---
def train(model, *datasets, epochs=10, lr=0.001, **kwargs):
    print(f"Training on {len(datasets)} datasets, {epochs} epochs")
    print(f"Extra params: {kwargs}")

train("ResNet", "train.csv", "val.csv", epochs=30, weight_decay=1e-4)

Lambda, map, filter, zip

Python
# lambda — anonymous one-liner
relu = lambda x: max(0, x)
relu(-3)   # 0
relu(5)    # 5

# map — apply function to every element (lazy)
losses = [0.8, 0.5, 0.3, 0.1]
rounded = list(map(lambda x: round(x, 2), losses))

# filter — keep elements passing predicate
pos = list(filter(lambda x: x > 0, [-1, 2, -3, 4]))  # [2, 4]

# zip — pair up iterables
features = [0.1, 0.9, 0.4]
labels   = [0,   1,   0  ]
for feat, lbl in zip(features, labels):
    print(f"feat={feat}, label={lbl}")

# Generator expression — memory-efficient
total = sum(x**2 for x in range(1_000_000))  # never materialises list

Closures & Decorators

Python
# Closure — inner function captures outer variable
def make_lr_scheduler(initial_lr):
    def decay(epoch):
        return initial_lr / (1 + 0.1 * epoch)
    return decay

scheduler = make_lr_scheduler(0.01)
scheduler(5)   # 0.00667

# Decorator — wraps a function to add behaviour
import time, functools

def timer(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        t0 = time.perf_counter()
        result = func(*args, **kwargs)
        print(f"{func.__name__} took {time.perf_counter()-t0:.4f}s")
        return result
    return wrapper

@timer
def train_epoch(data):
    time.sleep(0.1)   # simulate work

train_epoch([])   # train_epoch took 0.1001s
Section 04

Object-Oriented Programming

Most ML frameworks (PyTorch, sklearn) are class-based. Understanding OOP makes reading and writing framework code natural.

Python
class Layer:
    """Base class for neural network layers."""

    def __init__(self, name: str):
        self.name = name
        self.trainable = True

    def forward(self, x):
        raise NotImplementedError

    def __repr__(self):
        return f"Layer(name={self.name!r})"

    def __call__(self, x):        # makes instance callable: layer(x)
        return self.forward(x)


class LinearLayer(Layer):        # Inheritance
    import numpy as np

    def __init__(self, in_dim: int, out_dim: int):
        super().__init__("linear")
        import numpy as np
        # He initialisation — good for ReLU networks
        self.W = np.random.randn(in_dim, out_dim) * np.sqrt(2 / in_dim)
        self.b = np.zeros(out_dim)

    def forward(self, x):          # override parent method
        return x @ self.W + self.b  # matrix multiply + bias


layer = LinearLayer(4, 2)
print(layer)           # Layer(name='linear')

Memory Layout

An ndarray is a Python object wrapping a raw C buffer. Three key attributes govern everything:

Boolean (Mask) Indexing

Python
data = np.array([0.1, -0.5, 0.8, -0.2, 1.2])

mask = data > 0          # [True, False, True, False, True]
data[mask]               # [0.1, 0.8, 1.2]  — positive values only

# Compound mask
data[(data > 0) & (data < 1)]  # [0.1, 0.8]

# Use case: zero out negative activations (ReLU)
activations = np.random.randn(5)
activations[activations < 0] = 0   # in-place ReLU

# np.where — vectorised conditional
np.where(data > 0, data, 0)      # ReLU via where

Fancy (Integer Array) Indexing

Python
embeddings = np.random.randn(1000, 128)  # 1000 word vectors
token_ids  = np.array([5, 42, 17, 5])   # a sentence

sentence = embeddings[token_ids]         # shape (4, 128) — COPY

# 2-D fancy indexing — select specific (row, col) pairs
rows = np.array([0, 1, 2])
cols = np.array([2, 0, 1])
M[rows, cols]   # picks M[0,2], M[1,0], M[2,1]
⚠ Views vs Copies

Basic slices return views (same memory — fast, but mutations affect original). Boolean and fancy indexing return copies. Use arr.base is None or np.shares_memory(a, b) to check.

Mathematical Operations

NumPy implements the mathematical primitives that underpin every ML algorithm — from simple element-wise ops to full linear algebra solvers.

Element-wise Universal Functions (ufuncs)

Python
a = np.array([1.0, 4.0, 9.0, 16.0])
b = np.array([2.0, 2.0, 3.0,  4.0])

a + b        # [ 3.  6. 12. 20.]
a * b        # element-wise multiply
a ** 0.5    # [1. 2. 3. 4.] — square root
np.sqrt(a)  # same via ufunc
np.log(a)   # natural log
np.exp(a)   # e^x element-wise
np.abs(np.array([-1,2,-3]))   # [1 2 3]

# Aggregations
a.sum()         # 30.0
a.mean()        # 7.5
a.std()         # standard deviation
a.min(), a.max()
a.argmin(), a.argmax()  # index of min/max
np.cumsum(a)    # [1. 5. 14. 30.]

# Axis-wise aggregation on 2-D
M = np.array([[1,2,3],[4,5,6]])
M.sum(axis=0)   # [5 7 9]  — column sums
M.sum(axis=1)   # [6 15]  — row sums

Linear Algebra — the ML core

𝓔 Matrix multiply — the most important operation in ML

For matrices A ∈ ℝm×k and B ∈ ℝk×n, the product C = AB ∈ ℝm×n where each element Cij = Σl Ail Blj. In a neural network forward pass, every linear layer computes Y = XW + b — a batched matrix multiply.

Python — Linear Algebra
# Matrix multiply — three equivalent ways
A = np.random.randn(3, 4)
B = np.random.randn(4, 2)

C = A @ B                  # (3,2) — preferred syntax
C = np.dot(A, B)            # same
C = np.matmul(A, B)         # same for 2-D

# Element-wise vs matrix multiply — common bug!
A * B    # WRONG if shapes don't match — element-wise
A @ B    # CORRECT — matrix multiply

# Transpose
A.T                        # (4, 3) — view of A

# Dot product of two vectors
u = np.array([1, 2, 3])
v = np.array([4, 5, 6])
np.dot(u, v)               # 32  (= 1×4 + 2×5 + 3×6)

# L2 norm  ‖v‖ = √(Σ vᵢ²)
np.linalg.norm(v)          # 8.774

# Cosine similarity  cos θ = u·v / (‖u‖‖v‖)
cos_sim = np.dot(u,v) / (np.linalg.norm(u) * np.linalg.norm(v))

# Inverse and solve
A_sq = np.random.randn(3, 3)
np.linalg.inv(A_sq)         # A⁻¹
np.linalg.solve(A_sq, v)    # Ax = v → x (more stable than inv)

# Eigenvalues — used in PCA
eigenvalues, eigenvectors = np.linalg.eig(A_sq)

# SVD — the workhorse of dimensionality reduction
U, S, Vt = np.linalg.svd(A, full_matrices=False)
# A ≈ U @ np.diag(S) @ Vt

Broadcasting — Shape Magic

Broadcasting lets NumPy perform operations between arrays of different shapes without copying data. It is the single concept that makes vectorised code elegant and fast.

𝓔 Broadcasting Rules

Two shapes are compatible dimension-by-dimension from the right. A dimension is compatible if (1) the sizes are equal, or (2) one of them is 1. A size-1 dimension is "stretched" to match the other — conceptually, not in memory.

Python — Broadcasting in practice
# Normalise a batch: (batch, features) → subtract mean row
X = np.random.randn(100, 32)   # 100 samples, 32 features
mu = X.mean(axis=0)            # shape (32,) — feature means
std = X.std(axis=0) + 1e-8      # avoid /0
X_norm = (X - mu) / std        # (100,32)-(32,) → broadcasts ✓

# Pairwise Euclidean distances — clever broadcasting
# A: (m, d)   B: (n, d)  →  distances: (m, n)
A = np.random.randn(5, 3)
B = np.random.randn(4, 3)
diff = A[:, np.newaxis, :] - B[np.newaxis, :, :]  # (5,4,3)
dists = np.sqrt((diff**2).sum(axis=-1))             # (5,4)

# Outer product — (n,1) * (1,m) → (n,m)
u = np.array([1,2,3])[:, np.newaxis]  # (3,1)
v = np.array([10,20])[np.newaxis, :]  # (1,2)
u * v   # [[10,20],[20,40],[30,60]]

Performance & Vectorisation

Vectorisation is the art of replacing Python loops with NumPy operations. The speedup can be 100–1000×.

Loop vs Vectorised — Speed Comparison

Computing element-wise square of 1 million values:

~1200 ms

~2 ms
2 ms
Python — Refactor patterns
# ✗ SLOW — Python loop
result = []
for x in data:
    result.append(x ** 2)

# ✓ FAST — vectorised
result = data ** 2

# ✗ SLOW — manual dot product
dot = 0
for a, b in zip(u, v):
    dot += a * b

# ✓ FAST
dot = np.dot(u, v)

# ✗ SLOW — manual softmax
def softmax_slow(z):
    exp_z = [math.exp(x) for x in z]
    total = sum(exp_z)
    return [e / total for e in exp_z]

# ✓ FAST — numerically stable vectorised softmax
def softmax(z):
    z = z - z.max()          # subtract max for numerical stability
    e = np.exp(z)
    return e / e.sum()

Memory Tips

Python
# Check memory usage
arr = np.zeros((1000, 1000))
arr.nbytes                  # 8_000_000 bytes = 8 MB (float64)
arr.astype(np.float32).nbytes  # 4_000_000 bytes — half!

# out= parameter avoids allocating new array
result = np.empty_like(arr)
np.add(arr, arr, out=result)    # no extra allocation

# Contiguous memory = faster ops
C_arr = np.ascontiguousarray(arr.T)  # make C-contiguous copy
arr.flags['C_CONTIGUOUS']           # check
✓ Golden Rules of NumPy performance

1. No Python loops over array elements — every loop is 100× slower.
2. Use float32 for ML — half memory, same or faster computation.
3. Prefer views over copies — slices are free; fancy indexing always copies.
4. Check shape before every operation — 95% of bugs are shape mismatches.
5. Use np.linalg.solve not np.linalg.inv — more numerically stable.
