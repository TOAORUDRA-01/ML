Section 01

Linear Regression

The simplest supervised learning algorithm — fit a line (or hyperplane) through data by minimising squared errors. Everything in ML descends from this.

Linear Regression
Supervised Regression Parametric
Train
O(n·d²) or O(nd)
Predict
O(d)
Parameters
d+1 weights
Hyperparams
α (learning rate)

The Model

Given features x ∈ ℝᵈ, predict a scalar output ŷ ∈ ℝ. The model is a weighted sum of features plus a bias term.

Model Equation
ŷ = w₀ + w₁x₁ + w₂x₂ + … + wᵈxᵈ = wᵀx + b

Loss Function — Mean Squared Error

MSE Loss
L(w) = (1/n) Σᵢ (yᵢ − ŷᵢ)² = (1/n) ‖y − Xw‖²

Two Ways to Solve

1. Normal Equation (Closed-form, small d)

Normal Equation
w* = (XᵀX)⁻¹ Xᵀy

2. Gradient Descent (Large d, large n)

Gradient Descent Update
w ← w − α · ∇L = w − (2α/n) Xᵀ(Xw − y)

Interactive: Gradient Descent

// Watch the line converge
Python — Linear Regression
from sklearn.linear_model import LinearRegression
from sklearn.model_selection import train_test_split
from sklearn.metrics import mean_squared_error, r2_score
import numpy as np

# Generate toy data
np.random.seed(42)
X = np.random.randn(200, 3)
y = 3*X[:,0] - 2*X[:,1] + 1.5*X[:,2] + np.random.randn(200)*0.5

X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

# Fit
model = LinearRegression()
model.fit(X_train, y_train)

# Evaluate
y_pred = model.predict(X_test)
print(f"MSE: {mean_squared_error(y_test, y_pred):.4f}")
print(f"R²:  {r2_score(y_test, y_pred):.4f}")
print(f"Coefficients: {model.coef_}")  # [3, -2, 1.5]
print(f"Intercept:    {model.intercept_:.4f}")

# Normal equation — from scratch
X_aug = np.c_[np.ones(len(X_train)), X_train]  # add bias column
w_star = np.linalg.solve(X_aug.T @ X_aug, X_aug.T @ y_train)
# w_star ≈ model.intercept_ then model.coef_

Section 02

Logistic Regression

Despite the name, logistic regression is a classification algorithm. It models the probability of class membership using the sigmoid function.

Logistic Regression
SupervisedClassificationProbabilistic
Train
O(n·d·iter)
Predict
O(d)
Boundary
Linear
Output
Probability [0,1]

The Sigmoid Function

Sigmoid / Logistic
σ(z) = 1 / (1 + e⁻ᶻ) where z = wᵀx + b
The sigmoid maps any real z to (0,1). Default decision boundary at σ(z) = 0.5 → z = 0.

Loss — Binary Cross-Entropy

Binary Cross-Entropy (Negative Log-Likelihood)
L = −(1/n) Σᵢ [ yᵢ log(ŷᵢ) + (1−yᵢ) log(1−ŷᵢ) ]
✓ Why not MSE for classification?

MSE with sigmoid gives a non-convex loss — gradient descent can get stuck in local minima. Cross-entropy is convex and its gradient ŷ − y has a beautiful form identical to linear regression.

Python — Logistic Regression
from sklearn.linear_model import LogisticRegression
from sklearn.preprocessing import StandardScaler
from sklearn.pipeline import Pipeline
from sklearn.metrics import classification_report, roc_auc_score

# Always scale features for logistic regression
pipe = Pipeline([
    ('scaler', StandardScaler()),
    ('clf',    LogisticRegression(C=1.0, max_iter=1000,
                                    solver='lbfgs', random_state=42))
])
pipe.fit(X_train, y_train)

# Hard predictions
y_pred = pipe.predict(X_test)

# Soft probabilities
y_proba = pipe.predict_proba(X_test)[:, 1]  # P(class=1)

print(classification_report(y_test, y_pred))
print(f"ROC-AUC: {roc_auc_score(y_test, y_proba):.4f}")

# Multi-class — OvR or Multinomial
LogisticRegression(multi_class='ovr')        # one-vs-rest
LogisticRegression(multi_class='multinomial') # softmax

# Regularisation — C = 1/λ (smaller C = stronger regularisation)
LogisticRegression(C=0.01, penalty='l2')   # Ridge (default)
LogisticRegression(C=0.01, penalty='l1',   # Lasso — sparse
                   solver='liblinear')

K-Nearest Neighbours

The laziest algorithm — it memorises all training data and classifies new points by a vote among their k closest neighbours. Surprisingly powerful.

K-Nearest Neighbours
SupervisedNon-parametricLazy
Train
O(1)
Predict
O(n·d)
Memory
O(n·d)
Key param
k (neighbours)

Distance Metrics

MetricFormulaBest for
Euclidean√(Σ(xᵢ−yᵢ)²)Continuous, low-dimensional data
ManhattanΣ|xᵢ−yᵢ|Sparse data, high dimensions
Minkowski(Σ|xᵢ−yᵢ|ᵖ)^(1/p)General (p=2→Euclidean, p=1→Manhattan)
Cosine1 − (x·y)/(‖x‖‖y‖)Text, embeddings, high-dim

Interactive: K Effect on Decision Boundary

// Adjust K to see underfitting vs overfitting
⚠ The Curse of Dimensionality

In high dimensions, all points become approximately equidistant from each other — nearest neighbours are no more similar than random points. Always reduce dimensionality (PCA, feature selection) or scale features before using KNN.

Python — KNN
from sklearn.neighbors import KNeighborsClassifier
from sklearn.model_selection import cross_val_score

# Must scale — KNN is distance-based!
from sklearn.preprocessing import StandardScaler
scaler = StandardScaler()
X_sc = scaler.fit_transform(X_train)

# Find optimal k via cross-validation
k_range = range(1, 31)
cv_scores = [
    cross_val_score(KNeighborsClassifier(n_neighbors=k),
                    X_sc, y_train, cv=5).mean()
    for k in k_range
]
best_k = k_range[cv_scores.index(max(cv_scores))]

# Final model
knn = KNeighborsClassifier(
    n_neighbors=best_k,
    metric='euclidean',
    weights='distance'  # closer neighbours vote more
)
knn.fit(X_sc, y_train)

# KNN for regression
from sklearn.neighbors import KNeighborsRegressor
reg = KNeighborsRegressor(n_neighbors=5)  # predicts mean of k neighbours
Section 04

Decision Trees

A tree splits the feature space recursively using axis-aligned rules. Highly interpretable, but prone to overfitting — the building block for all ensemble methods.

Decision Tree
SupervisedNon-parametricInterpretable
Train
O(n·d·log n)
Predict
O(depth)
Key params
max_depth, min_samples
Split criterion
Gini / Entropy / MSE

Splitting Criteria

Gini Impurity (CART default)
Gini(S) = 1 − Σₖ pₖ²
Information Gain (ID3 / C4.5)
IG = Entropy(parent) − Σⱼ (|Sⱼ|/|S|) · Entropy(Sⱼ)
Entropy
H(S) = −Σₖ pₖ log₂(pₖ)

Tree Structure — Visual

Each internal node tests one feature. Leaf nodes predict the majority class (classification) or mean (regression). p = churn probability in leaf.
Python — Decision Tree
from sklearn.tree import DecisionTreeClassifier, export_text, plot_tree
import matplotlib.pyplot as plt

tree = DecisionTreeClassifier(
    max_depth=4,           # limit depth to prevent overfitting
    min_samples_leaf=20,  # each leaf must have ≥20 samples
    criterion='gini',      # or 'entropy'
    random_state=42
)
tree.fit(X_train, y_train)

# Feature importances (mean decrease in Gini)
importances = pd.Series(tree.feature_importances_, index=feature_names)
importances.sort_values().plot.barh()

# Text representation
print(export_text(tree, feature_names=feature_names))

# Visual tree
fig, ax = plt.subplots(figsize=(20, 8))
plot_tree(tree, feature_names=feature_names,
          class_names=['retain','churn'],
          filled=True, rounded=True, ax=ax)

# Cost-complexity pruning — find best alpha
path = tree.cost_complexity_pruning_path(X_train, y_train)
ccp_alphas = path.ccp_alphas
# cross-validate each alpha, pick best

Section 06

Ensemble Methods

Combine many weak learners to build a strong one. Random forests and gradient boosting consistently win tabular ML competitions.

Bagging — Random Forest

Bagging (Bootstrap AGGregatING): each tree trained on a random subset of data AND features → reduces variance.
Python — Random Forest & XGBoost
from sklearn.ensemble import RandomForestClassifier, GradientBoostingClassifier
from xgboost import XGBClassifier
import shap

# ─── Random Forest ───
rf = RandomForestClassifier(
    n_estimators=300,      # number of trees
    max_depth=None,         # grow fully (bias low)
    max_features='sqrt',    # features per split
    min_samples_leaf=5,
    oob_score=True,          # free validation on out-of-bag samples
    n_jobs=-1,               # use all CPU cores
    random_state=42
)
rf.fit(X_train, y_train)
print(f"OOB score: {rf.oob_score_:.4f}")  # free accuracy estimate

# ─── XGBoost (Gradient Boosting) ───
xgb = XGBClassifier(
    n_estimators=500,
    max_depth=6,
    learning_rate=0.05,     # shrinkage — smaller = more trees needed
    subsample=0.8,           # row sampling per tree
    colsample_bytree=0.8,   # feature sampling per tree
    reg_alpha=0.1,           # L1 regularisation
    reg_lambda=1.0,          # L2 regularisation
    eval_metric='logloss',
    early_stopping_rounds=20,
    random_state=42
)
xgb.fit(X_train, y_train,
       eval_set=[(X_test, y_test)], verbose=False)

# ─── SHAP — explain predictions ───
explainer = shap.TreeExplainer(xgb)
shap_values = explainer.shap_values(X_test)
shap.summary_plot(shap_values, X_test, feature_names=feature_names)

Support Vector Machines

SVMs find the maximum-margin hyperplane that separates classes. The kernel trick lets them handle non-linear boundaries without explicit feature expansion.

SVM Optimisation Objective
min ½‖w‖² subject to yᵢ(wᵀxᵢ + b) ≥ 1 ∀i

The Kernel Trick

KernelK(x,z)Use case
LinearxᵀzHigh-dim data (text, genomics)
RBF / Gaussianexp(−γ‖x−z‖²)General-purpose, non-linear data
Polynomial(xᵀz + r)ᵈImage classification
Sigmoidtanh(αxᵀz + c)Neural net approximation
Python — SVM
from sklearn.svm import SVC, SVR
from sklearn.model_selection import GridSearchCV

# Always scale before SVM — it's distance-based
from sklearn.preprocessing import StandardScaler
scaler = StandardScaler()
X_sc_train = scaler.fit_transform(X_train)
X_sc_test  = scaler.transform(X_test)

# Grid search over C and gamma
param_grid = {
    'C':     [0.1, 1, 10, 100],
    'gamma': ['scale', 'auto', 0.001, 0.01],
    'kernel':['rbf', 'linear']
}
svc = GridSearchCV(SVC(probability=True), param_grid,
                   cv=5, scoring='roc_auc', n_jobs=-1)
svc.fit(X_sc_train, y_train)
print(f"Best params: {svc.best_params_}")
print(f"Best CV AUC: {svc.best_score_:.4f}")

# C — regularisation: large C = less margin, fewer violations
# γ — RBF width: large γ = narrow Gaussian = more complex boundary

K-Means Algorithm

K-Means Objective
min Σₖ Σ_{xᵢ∈Cₖ} ‖xᵢ − μₖ‖²   (minimise within-cluster variance)
Python — Clustering
from sklearn.cluster import KMeans, DBSCAN
from sklearn.metrics import silhouette_score

# ─── K-Means — find optimal k via elbow / silhouette ───
inertias, sil_scores = [], []
for k in range(2, 11):
    km = KMeans(n_clusters=k, n_init=10, random_state=42)
    km.fit(X)
    inertias.append(km.inertia_)               # WCSS — elbow method
    sil_scores.append(silhouette_score(X, km.labels_))

best_k = sil_scores.index(max(sil_scores)) + 2

km_final = KMeans(n_clusters=best_k, n_init=20, random_state=42)
df['cluster'] = km_final.fit_predict(X)
print(f"Cluster centres:\n{km_final.cluster_centers_}")

# ─── DBSCAN ───
# eps: neighbourhood radius  min_samples: core point threshold
db = DBSCAN(eps=0.5, min_samples=10)
labels = db.fit_predict(X)
# label -1 = noise point
n_clusters = len(set(labels)) - (1 if -1 in labels else 0)
noise_pts  = (labels == -1).sum()
print(f"Clusters: {n_clusters}, Noise: {noise_pts}")

Python — Evaluation
from sklearn.model_selection import StratifiedKFold, cross_validate
from sklearn.metrics import (classification_report, confusion_matrix,
                              roc_auc_score, average_precision_score)

# Stratified K-fold (preserves class ratio in each fold)
cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)
results = cross_validate(
    model, X, y, cv=cv,
    scoring=['roc_auc', 'f1', 'precision', 'recall'],
    return_train_score=True
)
for metric, scores in results.items():
    print(f"{metric}: {scores.mean():.4f} ± {scores.std():.4f}")

# Full classification report
print(classification_report(y_test, y_pred,
      target_names=['retain','churn']))

# Confusion matrix
cm = confusion_matrix(y_test, y_pred)

# Threshold tuning — find precision/recall tradeoff
from sklearn.metrics import precision_recall_curve
prec, rec, thresholds = precision_recall_curve(y_test, y_proba)
# choose threshold where precision = 0.8
idx = np.argmin(np.abs(prec - 0.8))
optimal_thresh = thresholds[idx]
y_pred_tuned = (y_proba >= optimal_thresh).astype(int)

✓ The Practical ML Workflow

1. Baseline: logistic regression or linear model → establishes a floor.
2. Tree baseline: random forest with default params → usually beats linear.
3. Boost: XGBoost/LightGBM with early stopping → usually the winner.
4. Tune: Bayesian hyperparameter search (Optuna) on the winner.
5. Explain: SHAP values for feature importance and debugging.
6. Calibrate: if probabilities matter, use Platt scaling or isotonic regression.

Python — Benchmark all algorithms at once
from sklearn.linear_model import LogisticRegression
from sklearn.neighbors  import KNeighborsClassifier
from sklearn.tree       import DecisionTreeClassifier
from sklearn.ensemble   import RandomForestClassifier, GradientBoostingClassifier
from sklearn.svm        import SVC
from sklearn.pipeline   import Pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.model_selection import cross_val_score
import pandas as pd

models = {
    'Logistic Reg'  : Pipeline([('sc', StandardScaler()), ('m', LogisticRegression(max_iter=500))]),
    'KNN-7'         : Pipeline([('sc', StandardScaler()), ('m', KNeighborsClassifier(7))]),
    'Decision Tree' : DecisionTreeClassifier(max_depth=6, random_state=42),
    'Random Forest' : RandomForestClassifier(n_estimators=200, random_state=42),
    'Grad. Boost'   : GradientBoostingClassifier(n_estimators=200, random_state=42),
    'SVM-RBF'       : Pipeline([('sc', StandardScaler()), ('m', SVC(probability=True))]),
}

results = {}
for name, model in models.items():
    scores = cross_val_score(model, X, y, cv=5,
                              scoring='roc_auc', n_jobs=-1)
    results[name] = {'mean': scores.mean(), 'std': scores.std()}
    print(f"{name:18s} ROC-AUC: {scores.mean():.4f} ± {scores.std():.4f}")

benchmark_df = pd.DataFrame(results).T.sort_values('mean', ascending=False)
