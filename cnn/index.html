Section 01

Why CNNs Beat MLPs on Images

A raw MLP on a 224×224 image needs 150,000+ weights per neuron. CNNs exploit three inductive biases that make vision tractable.

The Three Key Inductive Biases

BiasWhat it meansHow CNNs encode it
Local connectivityNearby pixels are more related than distant onesSmall kernel (3×3, 5×5) sees only local patch
Weight sharingThe same edge detector works anywhere in the imageOne filter applied to every position — same weights
Translation equivarianceA cat shifted right is still a catConvolution commutes with translation

Parameter Count Comparison

A single CNN filter uses 9 weights regardless of image size. An MLP first-layer neuron needs one weight per pixel — doesn't scale.
★ Key insight

Because the same filter slides across every position, a CNN learns "is there an edge here?" once and applies that detector everywhere. The MLP must learn a separate edge detector for every position — catastrophically inefficient.

Section 02

The Convolution Operation

A filter (kernel) slides over the input, computing a dot product at each position. The output is a feature map showing where and how strongly the filter detected its pattern.

Mathematics

2D Convolution (single channel)
(I * K)[i,j] = Σₘ Σₙ I[i+m, j+n] · K[m,n]
Output feature map size
H_out = ⌊(H_in + 2·pad − kernel) / stride⌋ + 1
W_out = ⌊(W_in + 2·pad − kernel) / stride⌋ + 1
Parameter count per conv layer
params = C_out × (C_in × K × K + 1)
e.g. 64 filters, 3×3, 32 input channels = 64 × (32×9+1) = 18,496

Key Convolution Hyperparameters

ParamEffectCommon values
Kernel sizeLarger = wider receptive field, more params3×3 (default), 1×1, 5×5, 7×7
StrideStep size; larger stride = smaller output1 (no downsample), 2 (halve size)
PaddingPreserve spatial size; "same" padding keeps H,W0 (valid), 1 for 3×3 same

GroupsDepthwise: each input channel has own filter (MobileNet)1 (standard), C_in (depthwise)

1×1 Convolution — the Bottleneck

// 1×1 conv — channel projection

A 1×1 convolution mixes information across channels without touching spatial dimensions. It's a learned linear projection of the channel axis — used to reduce/increase channel count cheaply (bottleneck in ResNet, NiN, Inception). Cost: C_out × C_in × 1 × 1 vs C_out × C_in × K × K.

Python — Convolution in PyTorch
import torch, torch.nn as nn

# Standard conv block: Conv → BN → ReLU (the atomic unit of CNNs)
class ConvBNReLU(nn.Module):
    def __init__(self, c_in, c_out, k=3, stride=1, groups=1):
        super().__init__()
        pad = k // 2           # "same" padding for odd kernels
        self.block = nn.Sequential(
            nn.Conv2d(c_in, c_out, k, stride, pad, groups=groups, bias=False),
            nn.BatchNorm2d(c_out),   # BN after conv, before ReLU
            nn.ReLU(inplace=True)
        )
    def forward(self, x): return self.block(x)

# Depthwise separable convolution (MobileNet building block)
class DepthwiseSeparable(nn.Module):
    def __init__(self, c_in, c_out, stride=1):
        super().__init__()
        self.dw = ConvBNReLU(c_in, c_in,  k=3, stride=stride, groups=c_in) # depthwise
        self.pw = ConvBNReLU(c_in, c_out, k=1)                             # pointwise
    def forward(self, x): return self.pw(self.dw(x))

# Dilated convolution (DeepLab — larger receptive field, same resolution)
nn.Conv2d(64, 64, kernel_size=3, dilation=2, padding=2)

# Check output size: (H+2p-k)/s + 1
x = torch.randn(1, 3, 224, 224)     # (B, C, H, W)
conv = nn.Conv2d(3, 64, 3, stride=2, padding=1)
print(conv(x).shape)              # (1, 64, 112, 112)
Section 03

Pooling & Feature Maps

Pooling reduces spatial dimensions, increases receptive field, and provides a degree of translation invariance — without any learnable parameters.

MaxPool 2×2, stride=2: halves both H and W. Each output = maximum of 4 input values in that region.
Pooling typeOperationUse case
Max PoolingTake max of regionMost common; detects presence of feature
Average PoolingMean of regionSmoother; used in global average pooling
Global Avg PoolMean over entire spatial dimReplaces FC layers; reduces params drastically
Adaptive PoolPool to any output sizeHandle variable input sizes (PyTorch's default)
Strided ConvConv with stride=2Learnable downsampling (modern preference over pool)
Python — Pooling layers
import torch.nn as nn

# MaxPool: halve spatial dims
pool = nn.MaxPool2d(kernel_size=2, stride=2)

# Global Average Pooling — flatten spatially
gap = nn.AdaptiveAvgPool2d((1, 1))  # (B, C, H, W) → (B, C, 1, 1)

# Usage in classifier head (modern approach)
class ClassifierHead(nn.Module):
    def __init__(self, n_features, n_classes):
        super().__init__()
        self.gap  = nn.AdaptiveAvgPool2d(1)  # any H,W → 1×1
        self.drop = nn.Dropout(0.2)
        self.fc   = nn.Linear(n_features, n_classes)

    def forward(self, x):
        x = self.gap(x)          # (B, C, 1, 1)
        x = x.flatten(1)         # (B, C)
        x = self.drop(x)
        return self.fc(x)

ResNet Block — The Most Important Innovation

Python — ResNet Bottleneck Block
class Bottleneck(nn.Module):
    expansion = 4
    def __init__(self, in_ch, out_ch, stride=1):
        super().__init__()
        # 1×1 → 3×3 → 1×1 (cheaper than 3×3→3×3)
        self.conv1 = ConvBNReLU(in_ch, out_ch,    k=1)
        self.conv2 = ConvBNReLU(out_ch, out_ch,   k=3, stride=stride)
        self.conv3 = nn.Sequential(
            nn.Conv2d(out_ch, out_ch*self.expansion, 1, bias=False),
            nn.BatchNorm2d(out_ch*self.expansion)
        )
        # Skip connection may need projection if channels/size change
        self.skip = (nn.Sequential(
            nn.Conv2d(in_ch, out_ch*self.expansion, 1, stride=stride, bias=False),
            nn.BatchNorm2d(out_ch*self.expansion)
        ) if stride != 1 or in_ch != out_ch*self.expansion else nn.Identity())
        self.relu = nn.ReLU(inplace=True)

    def forward(self, x):
        out = self.conv3(self.conv2(self.conv1(x)))
        return self.relu(out + self.skip(x))  # F(x) + x
Section 05

Transfer Learning

Pre-trained models encode 1M+ images of feature knowledge. Fine-tune them on your task — often reaching SOTA with just hundreds of examples.

Strategies by Dataset Size

Your data sizeStrategyWhat to freeze
< 500 samplesFeature extraction onlyFreeze all conv layers; only train new head
500 – 5000Fine-tune last 1–2 blocksFreeze early layers; low LR for later layers
5000 – 50KFine-tune all layersNothing frozen; use small LR everywhere
> 50KFull training or fine-tuneCan train from scratch or fine-tune
Python — Transfer Learning with torchvision
import torchvision.models as models
import torch.nn as nn

# ─── Load pretrained EfficientNet ───
weights = models.EfficientNet_B0_Weights.IMAGENET1K_V1
model   = models.efficientnet_b0(weights=weights)

# ─── Strategy 1: Feature extraction ───
for param in model.parameters():
    param.requires_grad = False          # freeze everything
# Replace classifier head
num_ftrs = model.classifier[1].in_features
model.classifier = nn.Sequential(
    nn.Dropout(0.2),
    nn.Linear(num_ftrs, NUM_CLASSES)    # only this is trained
)

# ─── Strategy 2: Fine-tune last block ───
for name, param in model.named_parameters():
    param.requires_grad = False          # freeze all first
for param in model.features[-1].parameters():  # unfreeze last block
    param.requires_grad = True
for param in model.classifier.parameters():
    param.requires_grad = True

# ─── Differential LR (lower LR for older layers) ───
optimizer = torch.optim.AdamW([
    {'params': model.features[:4].parameters(),  'lr': 1e-5},
    {'params': model.features[4:].parameters(),  'lr': 5e-5},
    {'params': model.classifier.parameters(),     'lr': 3e-4},
], weight_decay=1e-2)

# ─── Preprocessing must match pretrained model's expectations ───
from torchvision import transforms
transform = weights.transforms()   # auto-applies correct normalisation
# Or manually:
transform = transforms.Compose([
    transforms.Resize(256),
    transforms.CenterCrop(224),
    transforms.ToTensor(),
    transforms.Normalize(mean=[0.485,0.456,0.406],
                         std=[0.229,0.224,0.225])  # ImageNet stats
])
✓ Practical rule of thumb

Always start with EfficientNet-B0 or ResNet-50 as your baseline. Unfreeze progressively — train the head first (3–5 epochs), then unfreeze more layers with a lower LR. Use torchvision or timm (PyTorch Image Models) for 500+ pretrained architectures.

IoU — Intersection over Union

IoU — core detection metric
IoU = Area(Pred ∩ GT) / Area(Pred ∪ GT)
Good detection: IoU ≥ 0.5 (PASCAL) or 0.75 (COCO)
IoU = orange overlap / total area of both boxes. Used in NMS and mAP computation.
Python — YOLO with ultralytics
from ultralytics import YOLO
import cv2

# ─── Load pretrained YOLOv8 ───
model = YOLO('yolov8n.pt')   # n=nano, s=small, m=medium, l=large, x=xlarge

# ─── Inference ───
results = model.predict('image.jpg', conf=0.25, iou=0.45)
for r in results:
    boxes  = r.boxes.xyxy     # (N,4) bounding boxes [x1,y1,x2,y2]
    scores = r.boxes.conf     # (N,) confidence scores
    labels = r.boxes.cls      # (N,) class indices

# ─── Fine-tune on custom dataset ───
# data.yaml:
#   path: /dataset
#   train: images/train
#   val:   images/val
#   names: {0: cat, 1: dog}
model.train(data='data.yaml', epochs=100,
            imgsz=640, batch=16, device=0)

metrics = model.val()
print(f"mAP@50: {metrics.box.map50:.4f}")
print(f"mAP@50-95: {metrics.box.map:.4f}")

# ─── IoU from scratch ───
def compute_iou(boxA, boxB):
    # boxes: [x1, y1, x2, y2]
    xA = max(boxA[0], boxB[0]); yA = max(boxA[1], boxB[1])
    xB = min(boxA[2], boxB[2]); yB = min(boxA[3], boxB[3])
    inter = max(0, xB-xA) * max(0, yB-yA)
    aA = (boxA[2]-boxA[0]) * (boxA[3]-boxA[1])
    aB = (boxB[2]-boxB[0]) * (boxB[3]-boxB[1])
    return inter / (aA + aB - inter + 1e-6)

U-Net — The Encoder-Decoder Architecture

U-Net: encoder downsamples (red), decoder upsamples (blue), skip connections (green) preserve spatial detail lost during pooling.
Python — Simple UNet block
class UNetBlock(nn.Module):
    def __init__(self, c_in, c_out):
        super().__init__()
        self.double_conv = nn.Sequential(
            ConvBNReLU(c_in,  c_out),
            ConvBNReLU(c_out, c_out)
        )
    def forward(self, x): return self.double_conv(x)

class UNet(nn.Module):
    def __init__(self, n_classes=2):
        super().__init__()
        self.enc1 = UNetBlock(3,   64)
        self.enc2 = UNetBlock(64,  128)
        self.enc3 = UNetBlock(128, 256)
        self.pool = nn.MaxPool2d(2)
        self.bottleneck = UNetBlock(256, 512)
        self.up2   = nn.ConvTranspose2d(512, 256, 2, stride=2)
        self.dec2  = UNetBlock(512, 256)  # 256 + 256 skip
        self.up3   = nn.ConvTranspose2d(256, 128, 2, stride=2)
        self.dec3  = UNetBlock(256, 128)
        self.up4   = nn.ConvTranspose2d(128, 64,  2, stride=2)
        self.dec4  = UNetBlock(128, 64)
        self.head  = nn.Conv2d(64, n_classes, 1)  # 1×1 final

    def forward(self, x):
        e1 = self.enc1(x)
        e2 = self.enc2(self.pool(e1))
        e3 = self.enc3(self.pool(e2))
        b  = self.bottleneck(self.pool(e3))
        d  = self.dec2(torch.cat([self.up2(b), e3], dim=1))
        d  = self.dec3(torch.cat([self.up3(d), e2], dim=1))
        d  = self.dec4(torch.cat([self.up4(d), e1], dim=1))
        return self.head(d)        # (B, n_classes, H, W)

from torchvision import transforms

# Training transforms — heavy augmentation
train_transform = transforms.Compose([
    transforms.RandomResizedCrop(224, scale=(0.5, 1.0)),
    transforms.RandomHorizontalFlip(),
    transforms.ColorJitter(brightness=0.4, contrast=0.4,
                            saturation=0.4, hue=0.1),

                         std=[0.229,0.224,0.225]),

Training Checklist

✓ Production CNN training checklist

✅ Start with pretrained weights (ImageNet) — always better than random init.
✅ Use RandAugment or AugMix for augmentation — avoids manual tuning.
✅ AdamW + cosine LR schedule with warmup (5–10% of total steps).
✅ Mixed precision (torch.cuda.amp) — 2× speed, half memory, same accuracy.
✅ timm library: 500+ pretrained models with consistent API.
✅ Monitor train vs val loss — first sign of overfitting = add augmentation or reduce model.
✅ Test Time Augmentation (TTA): average predictions on flipped/cropped versions at inference.

Python — Mixed precision + complete CV pipeline
import torch
from torch.cuda.amp import GradScaler, autocast
import timm

# ─── timm: 500+ models, consistent API ───
model = timm.create_model(
    'efficientnet_b0', pretrained=True,
    num_classes=NUM_CLASSES, drop_rate=0.2
).to(device)

# ─── Mixed precision training ───
scaler = GradScaler()
optimizer = torch.optim.AdamW(model.parameters(), lr=3e-4)

for xb, yb in train_loader:
    xb, yb = xb.to(device), yb.to(device)
    optimizer.zero_grad()

    with autocast():         # compute in float16
        logits = model(xb)
        loss   = criterion(logits, yb)

    scaler.scale(loss).backward()   # scaled backprop
    scaler.unscale_(optimizer)
    torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)
    scaler.step(optimizer)         # update in float32
    scaler.update()

# ─── Test Time Augmentation (TTA) ───
def tta_predict(model, image):
    model.eval()
    augments = [image, image.flip(-1)]   # orig + h-flip
    with torch.no_grad():
        preds = torch.stack([model(a.unsqueeze(0)) for a in augments])
    return preds.mean(dim=0)   # average logits
