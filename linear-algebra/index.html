Section 01

Vectors — Geometry & Operations

A vector is simultaneously a point in space, a direction with magnitude, and a column of numbers. All three views are useful in ML.

What is a Vector?

An n-dimensional vector v ∈ ℝⁿ is an ordered list of n real numbers. Geometrically it represents an arrow from the origin to a point.

(1.1) v = [v₁, v₂, …, vₙ]ᵀ  ∈  ℝⁿ
Vector addition forms a parallelogram. The diagonal is the resultant u+v.

Core Vector Operations

Addition
u + v = [u₁+v₁, u₂+v₂, …]ᵀ
Element-wise. Commutative & associative.
Scalar multiply
αv = [αv₁, αv₂, …]ᵀ
Scales magnitude; flips direction if α < 0.
Dot product
u·v = Σ uᵢvᵢ = ‖u‖‖v‖ cos θ
Scalar. Zero ↔ perpendicular vectors.
L2 Norm (length)
‖v‖₂ = √(Σ vᵢ²)
Euclidean distance from origin.
L1 Norm
‖v‖₁ = Σ |vᵢ|
Manhattan distance. Promotes sparsity (Lasso).
Cosine similarity
cos θ = (u·v) / (‖u‖‖v‖)
∈ [−1,1]. Used in NLP for semantic similarity.

Python — Vectors
import numpy as np

u = np.array([3.0, 4.0])
v = np.array([1.0, 0.0])

# Dot product
dot = np.dot(u, v)          # 3.0
dot_alt = u @ v            # same

# Norms
l2 = np.linalg.norm(u)     # 5.0  (√(9+16))
l1 = np.linalg.norm(u, 1) # 7.0  (3+4)

# Angle between vectors
cos_theta = dot / (np.linalg.norm(u) * np.linalg.norm(v))
angle_deg = np.degrees(np.arccos(np.clip(cos_theta, -1, 1)))
# clip avoids arccos(1.0000001) → NaN from float errors

# Unit vector (normalise)
u_hat = u / np.linalg.norm(u)  # ‖u_hat‖ = 1.0

# Projection of u onto v
proj = (np.dot(u, v) / np.dot(v, v)) * v
Section 02

Matrices — Structure & Transforms

A matrix is a linear transformation — it rotates, scales, reflects, and projects vectors. Every neural network layer is fundamentally a matrix operation.

Matrix Anatomy

Matrix A with shape (3,4). The main diagonal elements a₁₁, a₂₂, a₃₃ are highlighted.

Matrix Multiply — Geometry

The product C = AB where A ∈ ℝ^{m×k}, B ∈ ℝ^{k×n} gives C ∈ ℝ^{m×n}. Each element C_{ij} is the dot product of row i of A with column j of B.

(2.1) Cij = Σl=1k Ail · Blj
⚠ Shape rule — the most important rule in ML

(m×k) @ (k×n) → (m×n) — the inner dimensions must match. In a neural network with batch size B, input dim D, hidden dim H: X(B×D) @ W(D×H) = output(B×H).

Special Matrices

🎯
Identity I

Diagonal of 1s. AI = IA = A. The "do nothing" transform.

↔️
Transpose Aᵀ

Rows become columns. (AB)ᵀ = BᵀAᵀ.

🔄
Symmetric

A = Aᵀ. Covariance matrices are symmetric. Real eigenvalues.

🔁
Orthogonal

QᵀQ = I. Columns are orthonormal. Preserves lengths & angles.

📐
Diagonal

Only diagonal nonzero. Efficient to invert: D⁻¹ᵢᵢ = 1/Dᵢᵢ.

📉
Positive Definite

xᵀAx > 0 ∀x≠0. All eigenvalues > 0. Used in covariance.

Python — Matrices
import numpy as np

A = np.array([[1,2,3],[4,5,6]])   # shape (2,3)
B = np.array([[7,8],[9,10],[11,12]]) # shape (3,2)

# Matrix multiply: (2,3)@(3,2) → (2,2)
C = A @ B     # [[58,64],[139,154]]

# Transpose
A.T           # shape (3,2)

# Identity
I = np.eye(3) # 3×3 identity

# Inverse (square matrices only)
M = np.array([[2,1],[5,3]], dtype=float)
M_inv = np.linalg.inv(M)        # [[3,-1],[-5,2]]
M @ M_inv                        # ≈ identity

# Determinant
np.linalg.det(M)                 # 1.0 (2×3 - 1×5)

# Trace (sum of diagonal)
np.trace(M)                      # 5.0 (2+3)

# Frobenius norm (all-elements L2)
np.linalg.norm(A, 'fro')
Section 03

Core Linear Algebra

Span, rank, and systems of equations — the vocabulary for understanding what transformations can and cannot do.

Linear Independence & Span

📖 Definition — Linear Independence

Vectors v₁, v₂, …, vₖ are linearly independent if the only solution to α₁v₁ + α₂v₂ + … + αₖvₖ = 0 is α₁=α₂=…=αₖ=0. Intuitively: no vector is a "combination" of the others.

Rank & Null Space

ConceptDefinitionML Meaning
Rank(A)Number of linearly independent rows/columnsDegrees of freedom in a transformation
Full rankRank = min(m,n)No redundant features; invertible if square
Null spaceAll x where Ax = 0Directions that get destroyed by A
Column spaceAll vectors Ax can produceReachable outputs of a linear layer
DeterminantSigned volume scaling factordet=0 ↔ singular (non-invertible)

Solving Linear Systems: Ax = b

(3.1) Ax = b  →  x = A⁻¹b  (when A is invertible)
Python — Linear Systems
# Solve Ax = b  (prefer solve over inv — more numerically stable)
A = np.array([[3,1],[1,2]], dtype=float)
b = np.array([9, 8], dtype=float)

x = np.linalg.solve(A, b)   # [2. 3.]
# Verify: A @ x ≈ b

# Least squares (overdetermined: more equations than unknowns)
# Minimises ‖Ax - b‖²  →  normal equations: AᵀAx = Aᵀb
X_data = np.random.randn(100, 3)
y_data = np.random.randn(100)
coeffs, residuals, rank, sv = np.linalg.lstsq(X_data, y_data, rcond=None)

# Rank
np.linalg.matrix_rank(A)      # 2 (full rank)
Section 04

Eigenvalues, PCA & SVD

Eigendecomposition reveals the "natural axes" of a transformation. SVD generalises this to any matrix — it powers PCA, recommendation systems, and image compression.

Eigenvalues & Eigenvectors

📖 Definition

A non-zero vector v is an eigenvector of A with eigenvalue λ if: Av = λv. The matrix only scales the vector — does not rotate it. To find: solve det(A − λI) = 0 for λ, then (A − λI)v = 0 for each v.

A regular vector is rotated by A. An eigenvector only gets scaled by λ — it stays on the same line.
Python — Eigendecomposition
# Symmetric matrix → real eigenvalues, orthogonal eigenvectors
A = np.array([[4,1],[1,3]], dtype=float)

eigenvalues, eigenvectors = np.linalg.eig(A)
# eigenvalues  ≈ [4.618, 2.382]
# eigenvectors: each COLUMN is an eigenvector

# Verify: Av = λv
v0 = eigenvectors[:, 0]
A @ v0  # ≈ eigenvalues[0] * v0

# For symmetric matrices use eigh (faster, guaranteed real)
vals, vecs = np.linalg.eigh(A)

Singular Value Decomposition (SVD)

📖 SVD — The Most Useful Matrix Factorisation

Any matrix A ∈ ℝ^{m×n} can be written as A = UΣVᵀ, where U ∈ ℝ^{m×m} and V ∈ ℝ^{n×n} are orthogonal matrices, and Σ is diagonal with non-negative singular values σ₁ ≥ σ₂ ≥ … ≥ 0.

(4.1) A = UΣVᵀ  =  σ₁u₁v₁ᵀ + σ₂u₂v₂ᵀ + … + σᵣuᵣvᵣᵀ

Truncating to the top k terms gives the best rank-k approximation (Eckart-Young theorem) — the foundation of PCA and image compression.

Python — SVD & PCA
# ── SVD ──
A = np.random.randn(5, 4)
U, S, Vt = np.linalg.svd(A, full_matrices=False)
# U: (5,4)  S: (4,)  Vt: (4,4)

# Reconstruct from SVD
A_reconstructed = U @ np.diag(S) @ Vt   # ≈ A

# Low-rank approximation (keep top k singular values)
k = 2
A_approx = U[:, :k] @ np.diag(S[:k]) @ Vt[:k, :]

# ── PCA from scratch ──
X = np.random.randn(100, 10)    # 100 samples, 10 features
X_c = X - X.mean(axis=0)        # centre

# Covariance matrix approach
Cov = (X_c.T @ X_c) / (len(X_c) - 1)  # (10,10)
vals, vecs = np.linalg.eigh(Cov)
# Sort descending
idx = np.argsort(vals)[::-1]
PC = vecs[:, idx]                # principal components
X_2d = X_c @ PC[:, :2]          # project to 2D

# Explained variance ratio
explained = vals[idx] / vals.sum()
Section 05

Probability — Axioms & Rules

Probability is the language of uncertainty. Every ML model is secretly a probability machine — predicting likelihoods, not hard facts.

Kolmogorov Axioms

Non-negativity
P(A) ≥ 0
Probabilities are always ≥ 0.
Normalisation
P(Ω) = 1
Something must happen.
Additivity
P(A∪B) = P(A)+P(B) if A∩B=∅
Disjoint events add up.

Key Probability Rules

RuleFormulaWhen to use
ComplementP(Aᶜ) = 1 − P(A)Easier to compute "not A"
UnionP(A∪B) = P(A)+P(B)−P(A∩B)Non-mutually-exclusive events
ConditionalP(A|B) = P(A∩B)/P(B)Given B occurred, prob of A
Chain ruleP(A∩B) = P(A|B)·P(B)Joint from conditional
IndependenceP(A∩B) = P(A)·P(B)Events don't affect each other
Law of total prob.P(A) = Σ P(A|Bᵢ)P(Bᵢ)Marginalise over partition
Inclusion-exclusion principle — subtract the overlap to avoid double counting.
Python — Probability with NumPy
import numpy as np

# Simulate coin flips (Monte Carlo)
np.random.seed(42)
flips = np.random.randint(0, 2, size=10000)  # 0=T, 1=H
p_heads = flips.mean()                         # ≈ 0.5

# Conditional probability P(A|B)
# Disease: 1% prevalence. Test: 99% sensitivity, 95% specificity
n = 100_000
has_disease = np.random.random(n) < 0.01

test_pos = np.where(
    has_disease,
    np.random.random(n) < 0.99,   # true positive rate
    np.random.random(n) < 0.05    # false positive rate
)

# P(disease | positive test) — Bayes!
p_dis_given_pos = (has_disease & test_pos).sum() / test_pos.sum()
# ≈ 0.167 — surprising! Only 16.7% despite 99% accurate test
Section 06

Key Distributions

Distributions are the building blocks of probabilistic models. Every loss function, regulariser, and generative model assumes a distribution somewhere.

Discrete Distributions

Bernoulli
X ~ Bernoulli(p)
Single trial. Binary classification output neuron.
Binomial
X ~ Bin(n, p)
k successes in n trials. Dropout analysis.
Categorical
X ~ Cat(p₁…pₖ)
Multi-class output. Softmax produces this.

Continuous Distributions

Gaussian N(μ,σ²)
f(x) = (1/σ√2π)exp(−(x−μ)²/2σ²)
Weights init, noise, Central Limit Theorem.
Uniform U(a,b)
f(x) = 1/(b−a)
Random init, random search hypertuning.
Exponential Exp(λ)
f(x) = λe^{−λx}, x≥0
Time between events. Survival analysis.
Python — Sampling Distributions
import numpy as np
from scipy import stats

rng = np.random.default_rng(42)

# Gaussian: N(μ=0, σ=1)
samples = rng.normal(0, 1, 1000)

# PDF and CDF via scipy
x = np.linspace(-4, 4, 200)
pdf = stats.norm.pdf(x, loc=0, scale=1)
cdf = stats.norm.cdf(x)

# P(X > 2) for standard normal
p = 1 - stats.norm.cdf(2)   # ≈ 0.0228  (2-sigma tail)

# Multivariate Gaussian (for GDA, GMM)
mu    = np.array([0, 0])
sigma = np.array([[1, 0.8],[0.8, 1]])
mvn   = rng.multivariate_normal(mu, sigma, 500)

# Softmax — converts logits to categorical probs
def softmax(z):
    e = np.exp(z - z.max())
    return e / e.sum()

softmax(np.array([2.0, 1.0, 0.1]))  # [0.659, 0.242, 0.099]
Section 07

Bayes' Theorem & Inference

Bayes' theorem is the mathematical law of belief updating. It explains how to revise our prior beliefs when we observe new evidence.

(7.1) P(H|E) = P(E|H) · P(H) / P(E)
🧠
P(H) — Prior

Our belief about hypothesis H before seeing evidence.

📡
P(E|H) — Likelihood

How probable is the evidence if H were true?

📊
P(E) — Marginal

Total probability of seeing E under all hypotheses.

✅
P(H|E) — Posterior

Updated belief in H after observing evidence E.

Classic Example: Medical Test

Despite a 99% accurate test and a positive result, only 16.6% probability of disease. The low prior (1%) dominates.
⭐ ML Connection — Naive Bayes Classifier

Naive Bayes assumes feature independence: P(x₁,x₂,…,xₙ|y) = Π P(xᵢ|y). Then predict: ŷ = argmax_y P(y) · Π P(xᵢ|y). Despite the "naive" assumption, it works well for text classification (spam detection).

Section 08

Statistics — Inference & Estimation

Statistics is how we reason about populations from samples. MLE — the backbone of most ML training objectives — lives here.

Descriptive Statistics

Mean
μ = (1/n) Σᵢ xᵢ
Expected value. Sensitive to outliers.
Variance
σ² = (1/n) Σᵢ (xᵢ − μ)²
Average squared deviation.
Covariance
Cov(X,Y) = E[(X−μₓ)(Y−μᵧ)]
Joint variability. + = co-move.
Correlation
ρ = Cov(X,Y) / (σₓσᵧ)
Normalised ∈ [−1,1]. Scale-free.
Std deviation
σ = √σ²
Same units as data. ±1σ ≈ 68%.
Z-score
z = (x − μ) / σ
Standardise. Used in normalisation.

Maximum Likelihood Estimation (MLE)

MLE finds the parameter θ that makes the observed data most probable. It is the training objective of linear regression, logistic regression, and most neural networks.

(8.1) θ̂_MLE = argmax_θ   L(θ) = argmax_θ   Σᵢ log P(xᵢ | θ)
✓ MLE → Loss functions

· Gaussian likelihood → Mean Squared Error (MSE) loss.
· Bernoulli likelihood → Binary Cross-Entropy loss.
· Categorical likelihood → Cross-Entropy loss (softmax output).
Adding a Gaussian prior on θ → L2 regularisation (Ridge). Laplace prior → L1 (Lasso).

Python — Statistics
import numpy as np
from scipy import stats

data = np.random.randn(1000)

# Descriptive
mean  = data.mean()
var   = data.var(ddof=1)    # ddof=1 for sample variance
std   = data.std(ddof=1)
med   = np.median(data)
pct   = np.percentile(data, [25, 75])  # IQR

# Correlation matrix
X = np.random.randn(100, 4)
corr = np.corrcoef(X.T)   # (4,4) correlation matrix

# MLE for Gaussian: closed form
mu_mle    = data.mean()          # MLE mean
sigma_mle = data.std(ddof=0)    # MLE std (biased, divide by n)

# Log-likelihood of data under fitted Gaussian
log_lik = stats.norm.logpdf(data, mu_mle, sigma_mle).sum()

# Cross-entropy loss (negative log-likelihood for classification)
y_true  = np.array([1, 0, 1, 1])
y_pred  = np.array([0.9, 0.1, 0.8, 0.7])
ce_loss = -(y_true * np.log(y_pred) + (1-y_true) * np.log(1-y_pred)).mean()

Information Theory Essentials

Entropy H(X)
H(X) = −Σ p(x) log p(x)
Average information / uncertainty in X.
Cross-entropy H(p,q)
H(p,q) = −Σ p(x) log q(x)
The classification loss function.
KL Divergence
KL(p‖q) = Σ p log(p/q)
How different q is from p. Used in VAEs.
Mutual Information
I(X;Y) = H(X) − H(X|Y)
Shared info. Feature selection criterion.
Python — Information Theory
import numpy as np

def entropy(p):
    """Shannon entropy. p is a probability vector summing to 1."""
    p = p[p > 0]              # avoid log(0)
    return -(p * np.log2(p)).sum()

entropy(np.array([0.5, 0.5]))        # 1.0 bit — max uncertainty
entropy(np.array([1.0, 0.0]))        # 0.0 bit — certain
entropy(np.array([0.25]*4))         # 2.0 bits — 4 equal classes

def cross_entropy(y_true, y_pred, eps=1e-9):
    return -np.sum(y_true * np.log(y_pred + eps))

def kl_divergence(p, q, eps=1e-9):
    p, q = p + eps, q + eps
    return np.sum(p * np.log(p / q))

# Note: Cross-entropy = Entropy(p) + KL(p||q)
# Minimising CE is equivalent to minimising KL from true dist.
